tem algum software open source que possa me ajudar no registro e análise de dados quantitativos?

Vou separar as opções entre coleta/registro, análise estatística e visualização, priorizando ferramentas realmente abertas e adequadas à pesquisa acadêmica.

Sim. Para o seu perfil de pesquisa, minha principal indicação é o jamovi.

Melhor opção inicial: jamovi

O jamovi é um programa estatístico gratuito e de código aberto, com uma interface semelhante a uma planilha. Permite registrar variáveis, importar bases em CSV, Excel, SPSS, Stata e SAS, produzir tabelas, gráficos e executar análises estatísticas sem exigir programação. Ele funciona sobre a linguagem R e permite visualizar o código correspondente às análises, facilitando uma transição posterior para métodos mais avançados. O arquivo do projeto também pode reunir dados, configurações das análises e resultados, favorecendo a reprodutibilidade. [jamovi.org]

Para sua pesquisa sobre mortes violentas de crianças e adolescentes, ele serviria para:

  • registrar cada ocorrência, processo ou decisão como uma linha;
  • definir variáveis numéricas, nominais e ordinais;
  • gerar frequências e tabelas cruzadas;
  • comparar território, faixa etária, raça/cor, sexo, tipo de violência e período;
  • aplicar testes de associação;
  • construir regressões;
  • gerar gráficos e tabelas para a tese.

O R é uma linguagem e ambiente aberto voltado à computação estatística e à produção de gráficos; o RStudio Desktop possui edição gratuita e aberta para trabalhar com scripts e projetos em R. [R Project]

O Baserow é interessante quando a pesquisa possui entidades relacionadas. Outras alternativas

GNU PSPP é uma alternativa livre para análise estatística de dados amostrais, particularmente útil para quem deseja uma ferramenta próxima da organização tradicional dos programas estatísticos. [GNU]

o python +sqlite3 resolveria tudo isso?

Python + SQLite3 pode resolver praticamente todo o ciclo de registro, tratamento e análise quantitativa da sua pesquisa, sobretudo se o trabalho for conduzido por você ou por uma equipe pequena. Contudo, não resolve tudo sozinho com a mesma facilidade de uma ferramenta pronta como jamovi, Baserow ou ODK.

O que essa combinação permite fazer

O SQLite funcionaria como banco de dados permanente. Nele você poderia armazenar, por exemplo:

  • ocorrências de mortes violentas;
  • vítimas;
  • processos judiciais;
  • decisões;
  • territórios;
  • instituições responsáveis;
  • categorias de análise;
  • fontes documentais;
  • histórico de alterações.

O Python faria a interface com o banco e permitiria:

  • importar planilhas e arquivos CSV;
  • limpar e padronizar dados;
  • identificar duplicidades;
  • validar campos;
  • realizar consultas;
  • calcular frequências, médias e proporções;
  • construir tabelas cruzadas;
  • executar testes estatísticos;
  • produzir gráficos;
  • exportar resultados para Excel, CSV, Word, HTML ou PDF;
  • automatizar rotinas repetitivas.

Exemplo aplicado à sua pesquisa

Em vez de manter tudo em uma única planilha, você poderia estruturar tabelas relacionadas.

Tabela de vítimas

sql
CREATE TABLE vitimas (
    id_vitima INTEGER PRIMARY KEY,
    sexo TEXT,
    raca_cor TEXT,
    idade INTEGER,
    faixa_etaria TEXT,
    regiao_administrativa TEXT
);

Tabela de ocorrências

sql
CREATE TABLE ocorrencias (
    id_ocorrencia INTEGER PRIMARY KEY,
    id_vitima INTEGER,
    data_ocorrencia TEXT,
    tipo_morte TEXT,
    meio_empregado TEXT,
    contexto TEXT,
    fonte TEXT,
    FOREIGN KEY (id_vitima) REFERENCES vitimas(id_vitima)
);

Tabela de decisões judiciais

sql
CREATE TABLE decisoes (
    id_decisao INTEGER PRIMARY KEY,
    tribunal TEXT,
    numero_processo TEXT,
    data_decisao TEXT,
    tema TEXT,
    menciona_prioridade_absoluta INTEGER,
    categoria_resultado TEXT
);

Isso evita repetições e permite consultas como:

sql
SELECT
    v.raca_cor,
    COUNT(*) AS total
FROM ocorrencias o
JOIN vitimas v ON v.id_vitima = o.id_vitima
GROUP BY v.raca_cor
ORDER BY total DESC;

Onde Python + SQLite3 é superior

Essa solução é especialmente forte em:

  • reprodutibilidade, porque todo procedimento pode ficar registrado em scripts;
  • automação, evitando refazer manualmente limpezas e cálculos;
  • controle metodológico, pois as regras de classificação ficam explícitas;
  • integração de bases, como dados policiais, judiciais, demográficos e territoriais;
  • jurimetria, com processamento de decisões e metadados;
  • análise espacial, caso você relacione os registros às regiões administrativas do DF;
  • auditabilidade, porque é possível conservar a base bruta e produzir uma base tratada separadamente.

O que não será tão simples

1. Digitação manual

O SQLite não possui, por padrão, uma interface amigável para preenchimento. Você precisaria:

  • inserir os dados por script;
  • importar uma planilha;
  • usar um programa como DB Browser for SQLite;
  • ou construir um formulário.

Para facilitar o registro, você poderia criar uma interface com:

  • Streamlit;
  • Flask;
  • Django;
  • PySide/PyQt.

Com Streamlit, por exemplo, seria possível criar um formulário no navegador para cadastrar cada ocorrência.

2. Análise estatística sem programação

No jamovi, você seleciona variáveis e testes por menus. Em Python, será necessário escrever e revisar código.

Isso oferece mais controle, mas exige:

  • conhecimento estatístico;
  • documentação adequada;
  • cuidado com tipos de variáveis;
  • interpretação correta dos testes.

3. Trabalho simultâneo em equipe

SQLite é excelente para uso local e individual, mas tem limitações quando várias pessoas precisam alterar o banco ao mesmo tempo.

Para colaboração intensa, seria mais adequado usar:

  • PostgreSQL;
  • MariaDB;
  • Baserow conectado a um banco;
  • uma aplicação web centralizada.

4. Formulários de coleta de campo

Python pode criar formulários, mas ferramentas como ODK já oferecem:

  • coleta offline;
  • validação de respostas;
  • sincronização;
  • gerenciamento de usuários;
  • formulários em dispositivos móveis.

Construir tudo isso do zero provavelmente não seria eficiente.

SQLite ou planilha?

Para uma base pequena e simples, uma planilha ainda pode ser suficiente. Mas o SQLite é superior quando existem:

  • diferentes tipos de entidades;
  • muitas variáveis;
  • relações entre tabelas;
  • necessidade de rastreabilidade;
  • grandes quantidades de registros;
  • rotinas de atualização;
  • risco de duplicação;
  • integração entre várias fontes.

Uma planilha única tende a gerar estruturas como esta:

text
vítima + ocorrência + processo + decisão + território + instituição

Tudo repetido na mesma linha. No SQLite, essas informações podem ser separadas e relacionadas por identificadores.

por exemplo, sobre estes dados que coletei, qual instrumento ou metodologia posso utilizar para triar quais são relevantes?

Vou transformar sua pergunta em um instrumento de triagem aplicável à base: critérios, pontuação e classificação dos dados entre núcleo analítico, contexto, apoio e descarte.

Sim. Para esses dados, eu utilizaria uma matriz de elegibilidade e relevância dos indicadores, construída a partir da pergunta de pesquisa. A triagem não deve ser feita apenas verificando se o dado “fala de violência”, mas se ele realmente mede o objeto definido.

No seu caso, a pergunta de triagem seria:

Este indicador contribui para identificar as características, os fatores de risco e a distribuição das mortes violentas de crianças e adolescentes no Distrito Federal e permite relacioná-las às prioridades da rede de proteção?

1. Primeiro: defina os limites operacionais

Antes de pontuar os dados, quatro decisões precisam ficar metodologicamente explícitas.

População

No plano jurídico, criança e adolescente correspondem, em princípio, à população de 0 a 17 anos. Entretanto, seus dados estão organizados nas faixas:

  • 0 a 4 anos;
  • 5 a 14 anos;
  • 15 a 19 anos;
  • 10 a 19 anos;
  • 15 a 29 anos.

Isso produz uma incompatibilidade importante: a categoria de 15 a 19 anos inclui pessoas de 18 e 19 anos, que não são adolescentes para os fins do Estatuto da Criança e do Adolescente. Portanto, esses dados podem ser utilizados, mas devem ser registrados como aproximações, e não como medidas exatas da população adolescente.

A categoria “jovens de 15 a 29 anos” é ainda menos adequada como indicador direto, pois mistura adolescentes e adultos. Os dados mostram, por exemplo, 89 homicídios de jovens de 15 a 29 anos em 2024, mas somente 13 homicídios na faixa de 15 a 19 anos. fileciteturn1file2

Território

O núcleo da pesquisa é o Distrito Federal. Assim:

  • dados do DF: potencialmente centrais;
  • dados do Brasil: comparativos ou contextuais;
  • dados por unidade da Federação: contextuais;
  • dados sem identificação territorial: baixa utilidade para a análise principal;
  • dados por região administrativa do DF: seriam os mais relevantes para a dimensão territorial.

Evento estudado

Você precisa decidir quais eventos integram a categoria “mortes violentas”:

  1. apenas homicídios;
  2. homicídios e suicídios;
  3. homicídios, suicídios e mortes decorrentes de intervenção estatal;
  4. mortes violentas intencionais;
  5. mortes violentas e evitáveis, incluindo sinistros de trânsito;
  6. homicídios registrados mais homicídios ocultos estimados.

Essa definição modifica completamente a relevância dos indicadores. Suicídios e lesões autoprovocadas, por exemplo, aparecem na base para pessoas de 10 a 19 anos, mas somente são centrais se a definição da pesquisa incluir mortes autoprovocadas. fileciteturn1file0

Período

Os dados de 2014 a 2024 formam uma série comparável. Contudo, o período da análise precisa coincidir com o recorte temporal do projeto ou ser justificado como uma etapa exploratória.

2. Instrumento recomendado

Sugiro uma triagem em duas etapas:

Etapa A — critérios eliminatórios

O indicador somente passa para a pontuação quando possui:

  1. fonte identificável;
  2. definição minimamente clara;
  3. ano ou período identificado;
  4. unidade de medida conhecida;
  5. território conhecido;
  6. grupo populacional identificável.

Quando um desses elementos estiver ausente, o dado não precisa ser apagado, mas deve ser classificado como pendente de validação.

Etapa B — matriz de relevância

Classificação: núcleo analítico.

Homicídios de 15 a 19 anos no DF

É provavelmente o dado de maior magnitude, mas inclui jovens de 18 e 19 anos. Deve ser utilizado com uma nota metodológica explícita e, preferencialmente, substituído por microdados que permitam selecionar somente 15, 16 e 17 anos. São relevantes porque a arma de fogo pode constituir mecanismo central da violência letal. Contudo, os dados apresentados abrangem toda a população do DF, sem discriminação etária. fileciteturn1file1

Medem morbidade, não mortalidade. Podem ajudar a interpretar os suicídios e as práticas de prevenção em saúde mental, mas não podem ser somadas aos óbitos. fileciteturn1file0

Indicadores contextuais

Homicídios gerais no DF

Os números e taxas gerais mostram o contexto de redução da violência letal no Distrito Federal. São úteis para comparar a tendência geral com a tendência entre crianças e adolescentes, mas não respondem diretamente à pergunta sobre essa população.

Classificação: contextual.

Uma análise útil seria verificar se a redução entre adolescentes foi:

  • proporcional à redução geral;
  • maior que a redução geral;
  • menor que a redução geral.

Homicídios de jovens de 15 a 29 anos

A faixa é excessivamente ampla para representar crianças e adolescentes. Pode mostrar o contexto da violência juvenil, mas não deve substituir os dados de adolescentes.

Classificação: contextual.

Homicídios de homens jovens

Também combinam adolescentes e adultos e não permitem identificar isoladamente meninos adolescentes.

Classificação: contextual.

Indicadores nacionais ou comparações entre unidades da Federação

A informação de que o DF apresentou uma das menores taxas estimadas de homicídio em 2024 ajuda a contextualizar o caso, mas não explica a distribuição interna nem as mortes de crianças e adolescentes.

Classificação: contextual introdutório.

Dados que devem ser excluídos da análise principal

Homicídios de idosos

Não correspondem à população estudada. Só seriam pertinentes em um desenho comparativo entre faixas etárias, o que não parece ser o objeto central.

Classificação: excluir da base analítica principal.

Violência contra idosos

Mede outra população e, em parte, outro desfecho, como notificações e internações.

Classificação: excluir.

Homicídios de mulheres em geral

Sem recorte etário, não permite analisar meninas e adolescentes.

Classificação: contextual ou excluir.

Homicídios de mulheres negras e não negras

Podem demonstrar desigualdade racial e de gênero no DF, mas não comprovam essa desigualdade entre crianças e adolescentes.

Classificação: contextual, até que exista recorte etário.

Homicídios de indígenas em geral

Os números são muito pequenos e não possuem recorte de idade. As taxas também podem ser extremamente instáveis porque a população indígena utilizada como denominador é pequena.

Classificação: contextual ou indicador de lacuna.

Violência contra pessoas com deficiência

O dado apresentado se refere a notificações de violência, não a mortes violentas, e sua formatação não permite identificar claramente numeradores, denominadores e categorias.

Classificação: pendente de validação.

Sinistros de transporte terrestre

São mortes violentas ou externas, mas geralmente não intencionais. Somente devem entrar se a definição da pesquisa abranger mortes evitáveis e não apenas mortes intencionais.

Classificação: excluir ou criar um módulo separado.

4. Lacunas também são resultados

Alguns dados não devem ser tratados como “irrelevantes”, mas como lacunas de informação.

É o caso da ausência de:

  • dados de crianças e adolescentes LGBTQIAPN+ no DF;
  • mortes de meninas negras por idade;
  • mortes de crianças e adolescentes indígenas;
  • mortes de crianças e adolescentes com deficiência;
  • dados por região administrativa;
  • dados combinando idade, raça, sexo e território;
  • dados sobre contexto da ocorrência;
  • dados sobre relação entre vítima e autor;
  • dados sobre atuação anterior da rede de proteção.

Nesse caso, a classificação correta é:

Relevante para a pesquisa, mas indisponível na fonte consultada.

Isso é diferente de “dado irrelevante”. A ausência pode demonstrar limitações institucionais de produção de informações e dificultar a própria definição de prioridades de proteção.

5. Problemas de consistência que precisam ser triados

Antes da análise estatística, sua base também precisa passar por uma auditoria de qualidade.

Identifiquei, no material, alguns exemplos:

Rótulo aparentemente incorreto

O item denominado “Número de homicídios por arma de fogo por 100 mil habitantes” apresenta valores como 633, 490 e 517. Esses valores parecem ser números absolutos, e não taxas por 100 mil habitantes. A linha seguinte é que apresenta as taxas, como 22,9 e 17,5. fileciteturn1file1

Quantidade diferente de observações

A série da proporção de homicídios por arma de fogo apresenta dez valores, embora o período de 2014 a 2024 exija onze observações.

Categorias etárias incompatíveis

Há séries de 0–4, 5–14, 15–19, 10–19 e 15–29. Elas não podem ser diretamente agregadas sem risco de sobreposição.

Diferença entre indicador absoluto e taxa

Números absolutos são importantes para dimensionar a quantidade de casos. Taxas são necessárias para comparar grupos e anos com populações diferentes. Os dois indicadores devem ser preservados, mas nunca tratados como equivalentes.

Pequenos denominadores

Taxas de homicídio indígena ou de grupos populacionais pequenos podem variar intensamente com apenas um ou dois casos. Nesses casos, devem ser apresentados também:

  • número absoluto;
  • tamanho da população;
  • média móvel;
  • intervalo de confiança, quando possível.

A metodologia mais adequada, portanto, não é selecionar os indicadores pela magnitude ou pelo fato de produzirem gráficos interessantes. É aplicar um protocolo explícito de elegibilidade, qualidade e aderência às perguntas de pesquisa. o que preciso estudar para elaborar um protocolo como esse para minha pesquisa

Vou organizar isso como uma trilha de formação aplicada ao seu projeto, distinguindo o que é essencial para construir o protocolo do que pode ficar para uma etapa posterior.

Para elaborar um protocolo defensável para sua pesquisa, você precisa estudar metodologia quantitativa aplicada a dados secundários, com ênfase em epidemiologia, qualidade da informação e operacionalização de conceitos. Não é necessário dominar regressões avançadas ou aprendizado de máquina antes de começar.

Seus dados misturam números absolutos, taxas, estimativas, notificações, internações e óbitos; diferentes faixas etárias; indicadores do DF e do Brasil; além de homicídios, suicídios, lesões autoprovocadas e sinistros de trânsito. 2. Epidemiologia e estatísticas de mortalidade

Como o objeto envolve óbitos e populações, você precisa dominar epidemiologia descritiva básica.

Estude:

  • número absoluto;
  • razão, proporção e percentual;
  • taxa bruta e taxa específica;
  • numerador e denominador;
  • população exposta ao risco;
  • mortalidade por causa;
  • taxa específica por idade, sexo e raça;
  • padronização de taxas;
  • séries temporais;
  • variação percentual;
  • média móvel;
  • intervalo de confiança;
  • instabilidade de taxas com pequenos números;
  • mudança de composição populacional.

Questões que você precisa aprender a responder

Uma queda de 165 para 13 homicídios entre pessoas de 15 a 19 anos é uma redução absoluta e percentual. Mas, para interpretar adequadamente, você também precisa saber:

  • a população dessa faixa diminuiu?
  • houve alteração na classificação dos óbitos?
  • a redução foi maior ou menor que na população geral?
  • o comportamento foi contínuo ou oscilante?
  • os dados de 2024 são preliminares ou consolidados?
  • quantos dos 13 registros eram de pessoas de 18 e 19 anos?
  • uma taxa baseada em dois óbitos deve ser comparada diretamente com outra baseada em centenas?

O Sistema de Informações sobre Mortalidade reúne dados quantitativos e qualitativos sobre os óbitos no país, tendo a Declaração de Óbito como documento fundamental. Por isso, conhecer o processo de produção desses registros é indispensável para interpretar os números, e não apenas calculá-los. [Serviços e Informações do Brasil]

3. Sistema de Informações sobre Mortalidade e causas externas

Você deve estudar especificamente como os dados brasileiros de mortalidade são produzidos.

Conteúdos essenciais:

  • estrutura da Declaração de Óbito;
  • causa básica, causas antecedentes e causas associadas;
  • classificação pela CID-10;
  • causas externas;
  • agressões;
  • lesões autoprovocadas;
  • eventos de intenção indeterminada;
  • local de ocorrência e local de residência;
  • investigação de óbitos;
  • completude dos campos;
  • redistribuição e estimação de causas;
  • diferença entre registro policial e registro sanitário.

Isso será decisivo para compreender os chamados homicídios ocultos. Eles não são simplesmente homicídios que estavam “escondidos” no banco: são estimativas produzidas a partir de mortes violentas classificadas inicialmente como de intenção indeterminada, mediante metodologia específica. A edição de 2026 do Atlas da Violência mantém análises separadas para homicídios registrados e homicídios estimados. [Repositório Ipea]

  5. documentação sobre mortes violentas por causa indeterminada.

4. Teoria da mensuração

A matriz anterior utiliza palavras como “adequação”, “relevância” e “qualidade”. Para que elas não se tornem avaliações subjetivas, você deve estudar mensuração.

Conceitos centrais:

  • construto;
  • indicador observável;
  • validade de conteúdo;
  • validade de construto;
  • validade de critério;
  • confiabilidade;
  • precisão;
  • sensibilidade;
  • especificidade;
  • erro de mensuração;
  • viés de classificação;
  • falso positivo e falso negativo;
  • validade interna e externa.

Aplicação

Considere o indicador:

Homicídios de pessoas de 15 a 19 anos.

Ele é confiável como medida da faixa registrada pela fonte, mas possui validade limitada para representar “adolescentes”, porque inclui pessoas de 18 e 19 anos.

Portanto:

  • o dado não é necessariamente ruim;
  • sua correspondência com o conceito jurídico é imperfeita;
  • a limitação precisa ser registrada;
  • ele não deve receber a mesma classificação de um indicador de 15 a 17 anos.

Essa distinção entre qualidade do dado e adequação do dado à sua pergunta deve estruturar todo o protocolo.

6. Pesquisa com dados secundários e administrativos

Os dados do SIM, Sinan, registros policiais e sistemas judiciais não foram necessariamente produzidos para responder à sua pergunta de doutorado. Foram coletados para finalidades administrativas, sanitárias, policiais ou judiciais.

Você precisa estudar:

  • processo gerador do dado;
  • finalidade original da coleta;
  • cobertura da base;
  • subnotificação;
  • seleção dos casos registrados;
  • mudanças administrativas;
  • diferenças entre ausência do fenômeno e ausência do registro;
  • linkage entre bases;
  • duplicidade;
  • dados faltantes;
  • codificação posterior;
  • revisão de registros;
  • versões preliminares e consolidadas.

O RECORD é especialmente importante porque foi desenvolvido como extensão do STROBE para pesquisas observacionais baseadas em dados de saúde coletados rotineiramente para finalidades não necessariamente científicas. [record-statement.org]

Distinção importante

Um indicador pode apresentar:

  • boa qualidade na fonte, mas baixa relevância para sua pergunta;
  • alta relevância conceitual, mas baixa qualidade ou disponibilidade;
  • boa relevância e qualidade, mas baixa comparabilidade temporal;
  • ausência de dados, que se transforma em achado institucional.

9. Inferência causal, caso você use “fatores de risco”

Essa parte é indispensável se a tese pretender afirmar que determinada condição produz, aumenta ou explica mortes.

Estude:

  • associação versus causalidade;
  • exposição e desfecho;
  • confundidores;
  • mediadores;
  • colisores;
  • causalidade reversa;
  • diagramas causais — DAGs;
  • viés ecológico;
  • controle estatístico;
  • temporalidade;
  • plausibilidade causal;
  • análise de sensibilidade.

Com dados agregados, geralmente é mais seguro escrever:

“fatores associados à distribuição das mortes”

do que:

“causas das mortes”.

Para afirmar causalidade, não basta mostrar que duas taxas aumentam ou diminuem simultaneamente.

10. Construção e validação de instrumentos

A matriz de pontuação que apresentei anteriormente deve ser entendida como um protótipo. Para transformá-la em instrumento metodologicamente robusto, estude:

  • construção de escalas;
  • elaboração de itens;
  • validade de conteúdo;
  • avaliação por especialistas;
  • teste-piloto;
  • concordância entre avaliadores;
  • confiabilidade interavaliador;
  • índice Kappa;
  • análise de sensibilidade dos pontos de corte;
  • técnica Delphi;
  • métodos multicritério de decisão.

Alerta sobre a pontuação

Somar pontos de 0 a 3 pode criar uma falsa aparência de objetividade. Um indicador nacional, sem recorte etário, não deveria entrar no núcleo analítico apenas porque alcançou muitos pontos em outros critérios.

O protocolo deve combinar:

  1. critérios obrigatórios;
  2. critérios de exclusão;
  3. avaliação qualitativa;
  4. pontuação auxiliar;
  5. justificativa escrita;
  6. possibilidade de revisão.

Caso você realmente mantenha pesos, notas e uma pontuação global, deve estudar construção de indicadores compostos, incluindo normalização, ponderação, agregação e análise de sensibilidade. O manual da OCDE e do Joint Research Centre enfatiza que dimensões e variáveis devem decorrer de um quadro teórico e que escolhas de pesos e agregação precisam ser justificadas. [OECD]

como seria uma matriz de operacionalização. de exemplo para outro tema de pesquisa

Uma matriz de operacionalização é o instrumento que transforma a pergunta teórica da pesquisa em elementos empiricamente observáveis. Ela conecta:

problema de pesquisa → conceitos → dimensões → indicadores → variáveis → fontes → técnicas de análise.

A seguir, uso um tema diferente do seu para demonstrar.

Exemplo: evasão escolar no ensino médio

1. Tema

Evasão escolar entre estudantes do ensino médio da rede pública.

2. Problema de pesquisa

Em que medida as condições socioeconômicas, familiares e escolares estão associadas à evasão de estudantes do ensino médio da rede pública do Município Alfa entre 2022 e 2025?

3. Hipótese

A evasão escolar é mais frequente entre estudantes submetidos à vulnerabilidade econômica, à necessidade de trabalhar, à distorção idade-série e a vínculos frágeis com a escola.

4. Unidade de análise

Cada estudante matriculado no ensino médio da rede pública municipal ou estadual localizada no Município Alfa.

A unidade de análise precisa ser definida porque determina o significado das linhas da base:

text
1 linha = 1 estudante

Caso a pesquisa trabalhasse apenas com escolas, a unidade seria diferente:

text
1 linha = 1 escola

Isso resultaria em uma pesquisa ecológica ou institucional, e não em uma análise individual dos estudantes.

Como interpretar as colunas

Pergunta específica

É uma decomposição da pergunta principal.

A pesquisa não deve começar procurando todos os dados disponíveis. Primeiro se define o que precisa ser respondido:

  • qual é a magnitude da evasão?
  • quais estudantes estão mais expostos?
  • quais condições antecedem a evasão?
  • quais respostas institucionais estão presentes?

Cada pergunta específica deve gerar uma ou mais linhas na matriz.

Conceito

É a categoria teórica ou analítica utilizada pela pesquisa, como:

  • evasão escolar;
  • vulnerabilidade socioeconômica;
  • pertencimento escolar;
  • trajetória escolar;
  • proteção institucional.

O conceito geralmente é mais abstrato do que aquilo que aparece diretamente na base.

Definição operacional

É a regra utilizada para reconhecer empiricamente o conceito.

Por exemplo, “vulnerabilidade econômica” é uma categoria ampla. Ela pode ser operacionalizada como:

  • renda familiar per capita;
  • inscrição no CadÚnico;
  • recebimento de benefício social;
  • insegurança alimentar;
  • desemprego dos responsáveis.

A definição operacional deve dizer precisamente quando um caso será classificado naquela categoria.

Dimensão

Conceitos complexos normalmente possuem mais de uma dimensão.

A vulnerabilidade social, por exemplo, pode envolver:

  • renda;
  • alimentação;
  • habitação;
  • trabalho;
  • acesso a serviços;
  • estrutura familiar.

A dimensão divide o conceito em partes analiticamente distinguíveis.

Indicador

É uma medida utilizada para representar uma dimensão do fenômeno.

Exemplo:

text
Conceito: vínculo escolar
Dimensão: participação
Indicador: percentual de frequência mensal

Outro indicador do mesmo conceito poderia ser:

text
Conceito: vínculo escolar
Dimensão: pertencimento
Indicador: escore de pertencimento à escola

Variável

É a forma concreta pela qual o indicador aparece na base de dados.

text
frequencia_percentual
escore_pertencimento
reprovacoes_anteriores
renda_per_capita

O nome da variável deve ser acompanhado de definição, tipo, categorias e regras de preenchimento em um dicionário de dados.

Forma de mensuração

Informa como a variável será registrada:

  • binária;
  • nominal;
  • ordinal;
  • numérica discreta;
  • numérica contínua;
  • data;
  • texto padronizado.

Por exemplo:

VariávelTipo

Fonte

Indica de onde virá o dado:

  • base administrativa;
  • questionário;
  • entrevista;
  • observação;
  • documento;
  • registro institucional;
  • base pública;
  • informação calculada pelo pesquisador.

A fonte deve ser suficientemente precisa. Em vez de indicar apenas “escola”, seria melhor registrar:

text
Sistema de gestão escolar, módulo Matrículas, extração realizada em 10/03/2026

Relação entre hipótese e variáveis

A matriz também ajuda a separar os papéis das variáveis.

Variável dependente

É o resultado que a pesquisa pretende explicar:

text
Evasão escolar:
0 = não evadiu
1 = evadiu

Variáveis independentes

São as condições cuja associação com o resultado será examinada:

  • renda per capita;
  • trabalho remunerado;
  • horas de trabalho;
  • distorção idade-série;
  • reprovações anteriores;
  • frequência;
  • violência escolar;
  • tempo de deslocamento.

Variáveis de controle

São incluídas porque podem estar relacionadas tanto às condições examinadas quanto à evasão:

  • idade;
  • sexo;
  • raça/cor;
  • série;
  • escola;
  • território;
  • turno;
  • composição familiar.

Variáveis institucionais

Representam ações ou características das escolas:

  • acompanhamento psicossocial;
  • reforço escolar;
  • busca ativa;
  • alimentação escolar;
  • tamanho das turmas;
  • disponibilidade de transporte.

Inclusão das limitações

Uma boa matriz deve registrar limitações desde o início.

IndicadorLimitação
Renda familiar informadaPossibilidade de subdeclaração ou desconhecimento
Frequência escolarNão informa, sozinha, a razão das faltas
Registro de violênciaPode haver subnotificação

Situação “abandono”Pode ser posteriormente corrigida para transferência

Isso evita que a análise atribua aos dados uma capacidade explicativa que eles não possuem.

Regras práticas para elaborar a matriz

Uma matriz consistente deve observar cinco regras:

  1. Cada indicador deve responder a uma pergunta da pesquisa.

Se não responde a nenhuma pergunta, provavelmente é apenas contextual ou dispensável.

  1. Cada conceito abstrato deve receber definição operacional.

Termos como vulnerabilidade, proteção, risco ou prioridade não podem permanecer indefinidos.

  1. O indicador deve ser compatível com a unidade de análise.

Não se deve atribuir diretamente a um estudante uma característica medida apenas no nível da escola ou do bairro.

  1. A fonte precisa efetivamente conter a variável.

Não basta que a informação seja teoricamente relevante; ela deve estar disponível ou ser coletável.

  1. As limitações devem ser registradas antes da análise.

Isso reduz decisões oportunistas feitas depois de conhecer os resultados.

Em síntese, a matriz de operacionalização funciona como a arquitetura empírica da pesquisa. Ela demonstra como as categorias teóricas serão convertidas em evidências, quais dados serão admitidos e quais conclusões cada indicador permite — ou não permite — sustentar.

em resumo, o que precisa ter um protocolo de triagem

Em resumo, um protocolo de triagem de dados precisa conter:

  1. Finalidade da triagem

Explicar para que os dados serão selecionados e quais perguntas de pesquisa devem ajudar a responder.

  1. Definições operacionais

Definir precisamente os principais termos da pesquisa, como população, território, período, evento e unidade de análise.

  1. Critérios de inclusão

Estabelecer quais características um dado deve possuir para entrar na análise, por exemplo:

  • recorte etário compatível;
  • referência ao Distrito Federal;
  • período definido;
  • fonte identificável;
  • relação com o objeto da pesquisa.
  1. Critérios de exclusão

Indicar quando um dado será retirado, como:

  • população fora do recorte;
  • indicador sem fonte;
  • duplicidade;
  • conceito incompatível;
  • ausência de numerador, denominador ou unidade de medida.
  1. Critérios de qualidade

Avaliar:

  • completude;
  • consistência;
  • comparabilidade;
  • atualidade;
  • rastreabilidade;
  • precisão;
  • estabilidade das taxas.
  1. Classificação dos dados

Separar os indicadores em categorias padronizadas, como:

  • núcleo analítico;
  • complementar;
  • contextual;
  • lacuna relevante;
  • pendente de validação;
  • excluído.
  1. Regras para casos problemáticos

Prever como tratar:

  • faixas etárias incompatíveis;
  • dados faltantes;
  • pequenas contagens;
  • taxas sem denominador;
  • estimativas;
  • mudanças metodológicas;
  • divergências entre fontes.
  1. Procedimento de decisão

Informar quem realiza a triagem, em qual ordem, com quais instrumentos e como serão resolvidas dúvidas ou divergências.

  1. Registro da justificativa

Cada inclusão, exclusão ou reclassificação deve ter uma justificativa escrita e verificável.

  1. Teste-piloto e revisão

Aplicar o protocolo inicialmente a uma amostra dos dados, identificar problemas e revisar os critérios antes da aplicação definitiva.

Uma estrutura mínima de registro seria:

IndicadorFontePopulaçãoTerritórioPeríodoRelação com a perguntaQualidadeClassificaçãoDecisãoJustificativa

A lógica central é:

definir o objeto → estabelecer critérios → avaliar a qualidade → classificar o dado → registrar e justificar a decisão.

A pontuação pode auxiliar, mas não deve substituir critérios obrigatórios e uma justificativa metodológica.
